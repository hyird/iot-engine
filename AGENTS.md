# iot-engine 协作规范

本文件适用于整个仓库。实现需求时必须遵守以下约束；若用户在当前任务中给出更具体的要求，以用户的最新要求为准。

## 工作区与变更范围

- 先检查 `git status`，保留用户或其他任务正在进行的改动，不回退、不覆盖、不顺手格式化无关文件。
- 只修改当前需求涉及的文件。遇到并行新增的模块时，除非当前需求明确覆盖该模块，否则不要扩大修改范围。
- 清理“未使用”代码前必须确认它不是尚未接入但有明确用途的契约、Schema、API 或功能；能接入的校验代码应接入，不能仅因静态扫描未引用就删除。

## 前端依赖与工具

- 使用 Bun，保持 `package.json` 与 `bun.lock` 一致。
- 用户要求升级依赖时，直接升级到最新稳定版本；升级后运行冻结锁文件安装、类型检查、lint 和生产构建。
- 删除依赖前必须确认源码、构建配置和运行时均无使用。

## 前后端校验

- 前端和后端必须使用一致的校验规则，包括必填、长度、取值范围、枚举、格式、数组数量、分页和 ID 约束。
- 前端 Schema 不能只定义不使用：表单提交层与 API 调用层都应接入校验。
- 修改任一端的校验时，必须同步核对另一端以及数据库字段限制。
- 编辑表单中的空值、可选字段和清空行为必须与后端实际解析语义一致。

## 时间与时区

- 数据库只允许存储 UTC 时间。PostgreSQL 数据库和连接会话时区必须固定为 `UTC`，时间字段使用 `TIMESTAMPTZ`，禁止把本地时间或无时区时间直接写入数据库。
- API 接收设备时间时，必须先按设备配置的时区解析并转换为 UTC 后再入库；API 返回的业务时间统一使用带 `Z` 或明确 UTC 偏移的 ISO 8601 格式。
- 每台设备必须独立保存 `timezone` 字段，新增和编辑设备时均可配置。设备时区只用于解析设备报文时间和前端展示转换，不得改变数据库或连接会话时区。

## 前端布局与组件

- 层级结构不要放进业务表格做树形表格。使用页面左侧的独立树形区域进行导航或筛选，右侧保持普通平面表格。
- 表格必须在表格内部滚动：配置纵向滚动和必要的横向滚动，页面主体、页头和分页区域不能被长表格整体撑开。
- 页面布局优先直接修改现有页面。除非用户明确要求，不要为了布局调整新建组件或新增组件文件。
- 复用现有 Ant Design 组件、页面容器和工具类，不为一次性布局创建额外抽象。
- 新增或修改表格页面时，同时检查固定表头、横向滚动、纵向滚动、窄屏布局和分页位置。
- 所有管理表单弹窗统一使用现有 `FormModal`，不得直接使用 `Modal` 创建另一套表单弹窗规格。
- 管理表单弹窗统一宽度和高度规则：默认宽度 720px，高度使用 `min(720px, 90dvh)`，任何情况下不得超过可视区域高度的 90%。
- 弹窗标题区和底部操作区固定，表单内容区独立滚动；内容再多也不能让整个弹窗超出可视区域。
- 表单统一使用纵向标签布局。并排字段必须使用一致的列宽、间距和顶部对齐，输入控件应占满各自字段宽度。

## 样式与主题

- `web/styles/index.css` 只允许保留框架必需导入、层声明和主题 token。
- 禁止在 `index.css` 中编写组件选择器、Ant Design 覆盖、页面专用样式、滚动条样式或其他组件级 CSS。
- 全局视觉调整使用 Ant Design `ConfigProvider` theme token 或 Tailwind `@theme` token。
- 页面局部布局使用现有 JSX 工具类和组件属性，不通过全局 CSS 覆盖组件内部结构。

## React 状态安全

- Zustand selector、Hook 默认值以及 effect 依赖中的数组和对象必须保持引用稳定；不要在 selector 或 effect 依赖路径中反复创建 `[]`、`{}`。
- `useEffect` 内调用 `setState` 时必须确认依赖不会因该状态更新而持续变化，避免 React 最大更新深度错误。
- React Router 页面必须提供用户友好的错误兜底，不暴露默认的开发者错误页面。

## 验证要求

- 前端改动至少运行：`bun run typecheck`、`bun run lint`、`bun run build` 和 `git diff --check`。
- 修改过的前端文件必须通过 Biome 格式检查；不要为了通过全仓格式检查而改写无关的并行文件。
- 后端头文件、服务配置或静态资源策略有改动时，运行 `cmake --build build`。
- 涉及静态资源发布时，确认当前服务实际返回新 bundle；入口 HTML 不得使用会长期指向旧哈希资源的缓存策略。
- 布局改动应尽可能在本地运行页面中验证控制台错误、表格滚动和响应式表现；如果认证阻止页面检查，应明确说明未覆盖的验证范围。
